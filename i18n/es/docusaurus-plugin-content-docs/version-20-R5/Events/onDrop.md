## Descripción

The `On Drop` event is sent once to the destination object when the mouse pointer is released over the object. Este evento es la segunda fase de la operación de arrastrar y soltar, en la que se realiza una operación en respuesta a la acción del usuario.

This event is not sent to the object if the drag was not accepted during the [`On Drag Over`](onDragOver.md) events. If you process the `On Drag Over` event for an object and reject a drag, the `On Drop` event does not occur. Thus, if during the `On Drag Over` event you have tested the data type compatibility between the source and destination objects and have accepted a possible drop, you do not need to re-test the data during the `On Drop`. Ya sabe que los datos son adecuados para el objeto de destino.
